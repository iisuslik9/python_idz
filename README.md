Вариант 4: База данных логистической компании
1. Структура базы данных:
• Warehouse (id, name, location, capacity) — склад компании.
• Shipment (id, tracking_number, weight, status, warehouse_id) — груз,
связь один ко многим с Warehouse (каждый груз находится на одном
складе).
• Driver (id, name, license_number) — водитель компании.
• ShipmentDriver (id, shipment_id, driver_id, delivery_date) — связь многие
ко многим между Shipment и Driver (отражает, какой водитель
доставляет груз и дату доставки).
2. Через SQLAlchemy ORM и через прямые SQL-запросы с использованием
DB API 2.0 реализуйте следующие операции:
• Методы добавления записей в таблицы (Warehouse, Shipment, Driver,
ShipmentDriver).
• Метод выборки всех грузов на определённом складе.
• Функцию добавления записи о назначении водителя на доставку груза.
• Методы обновления данных о грузе (например, изменение статуса или
склада).
• Методы удаления груза или склада.
• Метод вычисления общего количества доставок, выполненных каждым
водителем.
3. Создайте данные для тестирования работы:
• Добавьте не менее 3 складов, не менее 8 грузов, не менее 4 водителей,
не менее 10 связей между грузами и водителями (назначение доставок).
